## 二、具体改动

### 2.1 PM_REVIEW_MODULES（修改）

```typescript
// 旧
export const PM_REVIEW_MODULES = [
  "整体结构", "自我评价", "项目经历", "工作经历", "技能", "教育背景", "格式",
] as const;

// 新
export const PM_REVIEW_MODULES = [
  "整体结构", "基础信息", "自我评价", "教育经历", "工作经历", "项目经历", "格式",
] as const;
```

变更说明：
- 移除 `"技能"` → 技能相关内容合并到自我评价模块批阅
- `"教育背景"` → 重命名为 `"教育经历"`（与设计文档对齐）
- 新增 `"基础信息"`
- 调整顺序：按简历从上到下的自然阅读顺序排列

### 2.2 PM_REVIEW_ACTION_TYPES（修改）

```typescript
// 旧
export const PM_REVIEW_ACTION_TYPES = [
  "rewrite", "delete", "merge", "reorder", "format", "verify", "condense",

export const PM_REVIEW_ACTION_TYPES = [
```

变更说明：
- 新增 `"add"`：用于缺少核心模块/内容时建议添加

### 2.5 新增：结构识别相关

```typescript
/** 结构识别输出的单个模块信息 */
export interface ModuleInfo {
  sectionTitle: string;        // 原始区块标题
  normalizedModule: string;    // 归类名称
  textContent: string;         // 该模块的文本内容
  needsDeepReview: boolean;    // 是否需要深入批阅
  mayNeedSearch: boolean;      // 是否可能需要搜索验证
}

/** 结构识别完整输出 */
export interface StructureAnalysis {
  modules: ModuleInfo[];
  missingModules: string[];    // 缺少的核心模块
  redundantModules: string[];  // 冗余模块
}
```

## 三、注意事项

1. **向后兼容**：所有新增字段均为可选（`?`），不破坏已有代码
2. **PM_REVIEW_MODULES 修改会触发编译错误**：`pm-review-parser.ts` 中的 `MODULE_ALIASES` 映射表引用了旧值，需在 M6 中同步更新。开发 M1 时可先修改类型，M6 阶段再修复 parser
3. **不删除旧常量**：`NOTE_CATEGORIES` 等简历优化工具的类型保持不动
4. **导出位置**：所有新类型追加到文件末尾，与已有代码分隔开，加注释标明分界
