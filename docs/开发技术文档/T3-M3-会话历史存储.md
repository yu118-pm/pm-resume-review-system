## 一、职责

提供批阅会话的 CRUD 操作，基于本地文件系统存储。支撑两个核心场景：
1. **会话管理**：创建/列出/查询学员会话
2. **多轮批阅**：保存每次批阅结果，读取上次结果用于历史对比

---

## 二、存储方案

### 2.1 目录结构

```
{REVIEW_DATA_DIR}/                    # 默认 ./data/review-sessions
├── sessions.json                     # 会话索引
└── {sessionId}/
    ├── meta.json                     # 会话元信息
    └── reviews/
        ├── 1.json                    # 第1轮批阅
        ├── 2.json                    # 第2轮批阅
        └── ...
```

### 2.2 文件格式

**sessions.json**（索引文件）：
```json
[
  {
    "id": "s_20260326_001",
    "studentName": "张三",
    "createdAt": "2026-03-26T00:00:00.000Z",
    "lastReviewAt": "2026-03-26T01:30:00.000Z",
    "reviewCount": 2
  }
]
```

**{sessionId}/meta.json**：
```json
{
  "id": "s_20260326_001",
  "studentName": "张三",
  "createdAt": "2026-03-26T00:00:00.000Z",
  "lastReviewAt": "2026-03-26T01:30:00.000Z",
  "reviewCount": 2
}
```

**{sessionId}/reviews/1.json**：
```json
{
  "round": 1,
  "timestamp": "2026-03-26T00:10:00.000Z",
  "resumeText": "（完整简历文本）",
  "comments": [ /* PmReviewComment[] */ ]
}
```

## 四、关键实现细节

### 4.1 sessionId 生成规则

```typescript
function generateSessionId(): string {
  const now = new Date();
  const date = now.toISOString().slice(0, 10).replace(/-/g, "");
  const rand = Math.random().toString(36).slice(2, 6);
  return `s_${date}_${rand}`;
}
// 示例：s_20260326_a3f2
```

### 4.2 getDataDir

```typescript
function getDataDir(): string {
  return process.env.REVIEW_DATA_DIR || "./data/review-sessions";
}
```

### 4.3 ensureDir 工具函数

```typescript
import { mkdir } from "node:fs/promises";

async function ensureDir(dir: string): Promise<void> {
  await mkdir(dir, { recursive: true });
}
```

### 4.4 createSession 流程

```
1. generateSessionId()
2. 构造 ReviewSession 对象
3. ensureDir(sessionDir)
4. 写入 meta.json
5. 创建 reviews/ 子目录
6. 读取 sessions.json → 追加新 session → 写回
```

### 4.5 saveReviewResult 流程

```
1. 读取 meta.json 获取当前 reviewCount
2. round = reviewCount + 1
3. 构造 ReviewHistoryEntry
4. 写入 reviews/{round}.json
5. 更新 meta.json（reviewCount++, lastReviewAt）
6. 更新 sessions.json 中对应记录
```

### 4.6 getLatestReview 流程

```
1. 读取 meta.json 获取 reviewCount
2. 若 reviewCount === 0，返回 null
3. 读取 reviews/{reviewCount}.json
4. 返回 ReviewHistoryEntry
```

## 五、错误处理

| 场景 | 处理 |
|------|------|
| sessions.json 不存在 | 返回空数组 `[]`，不报错 |
| sessionId 目录不存在 | `getSession` 返回 `null` |
| reviews 目录为空 | `getLatestReview` 返回 `null` |
| 写入失败（磁盘满等） | 抛出原始错误，由上层捕获 |
| JSON 解析失败 | 打日志 + 返回空/null |

## 六、注意事项

1. **文件编码统一 UTF-8**：`writeFile(path, data, "utf-8")`
2. **JSON 可读性**：`JSON.stringify(data, null, 2)`
3. **并发安全**：单用户场景暂不需要文件锁；若后续多用户需加锁
4. **gitignore**：`data/review-sessions/` 必须加入 `.gitignore`
5. **resumeText 要保存**：用于多轮对比时了解学员修改了什么
6. **sessions.json 作为索引**：每次创建/更新会话时同步更新，避免需要遍历所有目录
