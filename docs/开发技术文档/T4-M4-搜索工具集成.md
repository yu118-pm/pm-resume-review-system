## 三、Provider 架构

通过环境变量 `SEARCH_API_PROVIDER` 切换实现：

```typescript
interface SearchProvider {
  searchCompany(companyName: string): Promise<CompanySearchResult | null>;
  searchIndustry(keyword: string, metric: string): Promise<IndustrySearchResult | null>;
}

function getProvider(): SearchProvider {
  const provider = process.env.SEARCH_API_PROVIDER || "mock";
  switch (provider) {
    case "mock":
      return mockProvider;
    case "tavily":
      return tavilyProvider;  // 后续实现
    default:
      return mockProvider;
  }
}
```

导出函数内部委托给 provider：

```typescript
export async function searchCompany(companyName: string) {
  const provider = getProvider();
  try {
    return await withTimeout(provider.searchCompany(companyName), 5000);
  } catch {
    console.warn("[pm-review-tools] searchCompany 失败", { companyName });
    return null;
  }
}
```

## 五、超时与缓存

### 5.1 超时控制

```typescript
function withTimeout<T>(promise: Promise<T>, ms: number): Promise<T> {
  return Promise.race([
    promise,
    new Promise<never>((_, reject) =>
      setTimeout(() => reject(new Error(`搜索超时 (${ms}ms)`)), ms),
    ),
  ]);
}
```

每次搜索最多等待 **5 秒**。

### 5.2 内存缓存

同一次 Agent 调用周期内，相同查询不重复请求：

```typescript
export function createSearchCache() {
  const cache = new Map<string, unknown>();
  
  return {
    async searchCompany(name: string) {
      const key = `company:${name}`;
      if (cache.has(key)) return cache.get(key) as CompanySearchResult | null;
      const result = await searchCompany(name);
      cache.set(key, result);
      return result;
    },
    async searchIndustry(keyword: string, metric: string) {
      const key = `industry:${keyword}:${metric}`;
      if (cache.has(key)) return cache.get(key) as IndustrySearchResult | null;
      const result = await searchIndustry(keyword, metric);
      cache.set(key, result);
      return result;
    },
  };
}
```

Agent 引擎每次 `reviewResume()` 调用时创建一个新缓存实例。

## 六、注意事项

1. **搜索层不做判断**：只返回原始信息，判断逻辑交给 LLM（数据验证 Prompt）
2. **失败返回 null**：不抛错，Agent 层降级处理
3. **Mock 数据要合理**：覆盖常见互联网公司和行业指标，便于开发测试
4. **日志**：记录每次搜索的关键词和结果（成功/失败/缓存命中）
5. **后续扩展**：接入 Tavily 时只需实现 `tavilyProvider`，不改动接口
