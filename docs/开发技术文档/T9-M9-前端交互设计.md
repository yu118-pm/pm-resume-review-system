## 三、各组件详细设计

### 3.1 SessionSelector

**功能**：
- 展示已有会话列表（学员名、批阅次数、最后批阅时间）
- 创建新会话（输入框 + 按钮）
- 选择会话后进入上传阶段

**API 调用**：
- `GET /api/review-sessions` → 加载列表
- `POST /api/review-sessions` → 创建新会话

**UI 要素**：
- 会话卡片列表（学员名、reviewCount badge、lastReviewAt）
- "新建会话" 输入框 + 按钮
- 空状态提示

### 3.2 ResumeUploader

**功能**：
- 文件拖拽/选择上传（PDF/DOCX）
- 显示当前会话信息
- "开始批阅" 按钮
- Loading 状态

**流程**：
```
1. 用户上传文件
2. POST /api/parse-file → 获取 resumeText
3. 用户点击"开始批阅"
4. POST /api/review-pm-resume { resumeText, sessionId }
5. 等待返回 → 展示结果
```

**Loading 状态**：
- 显示旋转动画 + "正在批阅，请稍候...（预计 30-60 秒）"
- 禁用上传和按钮

### 3.3 ReviewResult

**功能**：
- 按 `normalizedModule` 分组展示批注
- 每组显示模块名 + 批注数量
- 展开/折叠模块
- 导出 Word 按钮

**分组逻辑**：
```typescript
const grouped = comments.reduce((acc, comment) => {
  const module = comment.normalizedModule || "其他";
  if (!acc[module]) acc[module] = [];
  acc[module].push(comment);
  return acc;
}, {} as Record<string, PmReviewComment[]>);
```

**模块排列顺序**：按 `PM_REVIEW_MODULES` 定义的顺序排列。

### 3.4 CommentCard

**功能**：
- 展示单条批注的所有信息
- 多轮状态颜色标注
- confidence 可视化
- searchEvidence 折叠展示
- needsConfirmation 标记

**多轮状态颜色**：

| previousRoundStatus | 颜色 | 标签 |
|---------------------|------|------|
| `new` | 蓝色 `bg-blue-50 border-blue-200` | 🆕 新发现 |
| `modified` | 橙色 `bg-orange-50 border-orange-200` | ✏️ 已修改仍需改进 |
| `unchanged` | 红色 `bg-red-50 border-red-200` | ⚠️ 上次已指出未修改 |
| `resolved` | 不显示（被 Guard 过滤） | — |
| 无状态 | 默认白色 | — |

**confidence 展示**：

| confidence | 样式 |
|-----------|------|
| `high` | 实心圆 `●` 绿色 |
| `medium` | 半实心 `◐` 黄色 |
| `low` | 空心圆 `○` 灰色 |

**卡片布局**：

```
┌────────────────────────────────────────────┐
│ [模块标签] [多轮状态标签] [confidence ●]    │
│ 📍 {location}                              │
│                                            │

│                                            │
```

postcss.config.mjs:

```js
const config = {
  plugins: {

  },
```

全局 CSS 中添加：
```css

```

### 4.3 使用范围

- **新 PM 批阅组件**：全部使用 Tailwind
- **已有组件**：保持现有 inline style 不动，不做迁移

## 五、导出 Word 流程

```typescript
async function handleExport() {
  if (!uploadedFile || comments.length === 0) return;
  
  setExporting(true);
  const formData = new FormData();
  formData.append("file", uploadedFile);
  formData.append("comments", JSON.stringify(comments));
  
  const response = await fetch("/api/export-pm-review-docx", {
    method: "POST",
    body: formData,
  });
  
  const blob = await response.blob();
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `批阅_${uploadedFile.name}`;
  a.click();
  URL.revokeObjectURL(url);
  setExporting(false);
}
```

## 六、注意事项

1. **前端最后开发**：等 M7 所有 API 就绪后再开始
2. **先做最小可用版本**：上传 → 批阅 → 结果展示 → 导出。会话管理可先做简化版
3. **不做复杂状态管理**：React state 足够
4. **Tailwind 只用于新组件**：不迁移已有 inline styles
5. **Loading 用简单动画**：不做 SSE 流式进度
6. **响应式不是重点**：桌面端使用为主，不需要移动端适配
