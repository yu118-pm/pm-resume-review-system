预估：0.5 天

### 模块 M2：分模块 Prompt 体系
**优先级：P0（Agent 核心能力）**

改动范围：
- `src/lib/pm-review-prompts.ts` → **重写**
  - 通用 System Prompt（共享基座）
  - 结构识别 Prompt
  - 基础信息批阅 Prompt
  - 自我评价批阅 Prompt
  - 教育经历批阅 Prompt
  - 工作经历批阅 Prompt
  - 项目经历批阅 Prompt（★核心）
  - 格式规范批阅 Prompt
  - 数据验证 Prompt（结合搜索结果）
  - 多轮一致性检查 Prompt
  - User Prompt 模板生成函数

参考文档：`AI处理与Prompt设计.md` 第六节

依赖：M1
预估：1 天

---

### 模块 M3：会话历史存储
**优先级：P0（多轮批阅基础）**

改动范围：
- 新建 `src/lib/pm-review-history.ts`
  - 存储方案：本地 JSON 文件（`data/review-sessions/`）
  - `createSession(studentName)` → sessionId
  - `listSessions()` → ReviewSession[]
  - `getReviewHistory(sessionId)` → ReviewHistoryEntry[] | null
  - `saveReviewResult(sessionId, resumeText, comments)` → void
  - `getLatestReview(sessionId)` → { comments, resumeText } | null

存储结构：
```
data/review-sessions/
  ├── sessions.json          # 会话列表
  └── {sessionId}/
      ├── meta.json           # 会话元信息
      └── reviews/
          ├── 1.json          # 第1次批阅（含 resumeText + comments + timestamp）
          └── 2.json          # 第2次批阅
```

依赖：M1
预估：0.5 天

---

### 模块 M4：搜索工具集成
**优先级：P1（数据验证增强）**

改动范围：
- 新建 `src/lib/pm-review-tools.ts`
  - `searchCompany(companyName)` → { name, industry, scale, mainBusiness }
  - `searchIndustry(keyword, metric)` → { benchmark, source, confidence }
  - 搜索实现：调用外部搜索 API（如 SerpAPI / Tavily / 自建）
  - 搜索失败降级处理
  - 结果缓存（同一会话内相同查询不重复搜索）

注意：需要选定搜索服务提供商，配置 API Key

依赖：M1
预估：0.5 天

---

### 模块 M5：Agent 编排引擎（★核心）
**优先级：P0**

改动范围：
- 新建 `src/lib/pm-review-agent.ts`
  - `reviewResume(resumeText, sessionId)` → 主函数
  - Step 1: 读取会话历史
  - Step 2: 结构识别（LLM 调用）
  - Step 3: 分模块批阅（循环调用各模块 Prompt）
  - Step 4: 数据验证（自主决策是否搜索）
  - Step 5: 历史对比（多轮场景）
  - Step 6: Guard 校验 + 汇总
  - Step 7: 保存结果

  关键设计：
  - 每步 LLM 调用独立 try-catch，单模块失败不阻塞整体
  - 搜索步骤可选跳过
  - 支持流式进度回调（告知前端当前处理到哪个模块）

依赖：M1, M2, M3, M4
预估：1.5 天

## 五、开发约定

### 5.1 代码规范
- TypeScript 严格模式
- 每个新模块必须有完整的类型定义
- 错误处理：每个 LLM 调用和外部 API 调用必须 try-catch
- 日志：关键步骤需记录日志（console.log，后续可替换为正式 logger）

### 5.2 测试策略
- M2（Prompt）：用 2-3 份真实简历手动测试各模块 Prompt
- M5（Agent）：端到端测试，验证完整流程
- M6（Guard）：单元测试，验证各校验规则
- M9（前端）：手动走通完整用户流程

### 5.3 环境变量
- `OPENAI_API_KEY`：已有
- `SEARCH_API_KEY`：新增（搜索服务 API Key）
- `SEARCH_API_PROVIDER`：新增（tavily / serpapi / mock）
- `REVIEW_DATA_DIR`：新增（会话数据存储目录，默认 `./data/review-sessions`）
